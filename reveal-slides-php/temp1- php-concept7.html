Port

Quoi ?

C’est comme une porte dans une maison.

Il permet aux données d’entrer et de sortir.

Chaque application a son propre port.

Port

Qui ?

Les applications web.

Les serveurs web.

Les clients web.

Port

Où ?

Dans les communications réseau.

Dans le code de votre application.

Dans la configuration de votre serveur.

Port

Quand ?

Lorsque vous voulez communiquer sur le réseau.

Lorsque vous configurez votre serveur.

Lorsque vous écrivez du code pour une application web.

Port

Comment ?

En l’ouvrant dans votre serveur.

En le spécifiant dans votre code.

En le fermant lorsque vous avez terminé.

Port

Combien ?

Il y a plus de 65 000 ports possibles.

Certains sont réservés pour des services spécifiques.

D’autres peuvent être utilisés librement.

Port

Pourquoi ?

Pour permettre aux applications de communiquer.

Pour diriger le trafic vers la bonne application.

Pour sécuriser les communications en n’ouvrant que les ports nécessaires.
